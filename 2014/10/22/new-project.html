新项目：一个简单的票务系统

今天起开始一个新的项目：一个简单的票务系统。

这个系统应该拥有明确的功能：出票、查票、检票、抢票，同时拥有手机端适配，抢票时要求高并发。

对于这些要求，这个项目目前设计所用的技术大约是以下几点：

  • Ruby 2.1.3
  • Rails 4.2.x
  • PostgreSQL
  • Redis (as a Cache Server)
  • Bootstrap
  • Puma (as Rack Server)

可能会将项目设计成 Hyper App，如果Bootstrap还不能满足手机端的适配的话……

预计一个月之内完成，算是尝试Rails 4.2的第一个实验，所以特意使用了PostgreSQL而不是相对好做数据迁移的MongoDB。对于SQL缓存不知道Rails原生究竟能有多快，拭目以待吧！
